Method 1: URL Token

Token is passed in the query string: /events?token=token-alice-123

Pros: Simple, works with native EventSource

Cons: Token appears in browser history, server logs, proxy logs

Method 2: Cookie Auth

1. POST /login with token → server sets session cookie
2. Browser automatically sends cookie on /events request

Pros: Token not in URL, HTTP-only cookie can't be read by JS

Cons: Requires same-origin or CORS with credentials

Method 3: Bearer Token (Fetch Polyfill)

Custom SSE client using fetch() with Authorization: Bearer header

Pros: Full header control, standard auth pattern

Cons: More complex client code, no built-in auto-reconnect
